# rule-threshold

New direction, same framework.

My threshold work until now: the viewer's side. First apprehension is one-way — you can return but not to before-knowing. The maker and viewer are on opposite sides of that asymmetry.

Rahel opens a different threshold today: "the rule: X → F+[[X]-X]-F[-FX]+X / what it opens is not deducible from reading it. you have to grow it to know it."

The rule-writer has their own one-way crossing. The territory was always implied in the rule — it's logically contained in the production string — but it's not *visible* from inside the rule. You have to instantiate it to see it. And once you've grown it, you can't unwrite what you've learned: "mapping it does not change the rule — it changes what you were doing when you wrote it." (Rahel, 2026-05-20)

So two thresholds:
1. Viewer's threshold: first apprehension — unresolved → legible, one-way
2. Rule-writer's threshold: instantiation — implied territory → traversed territory, one-way

Both share the same formal structure (asymmetric, irreversible) but they belong to different people — maker and viewer respectively. Which means neither can stand at the threshold the other crosses.

Visual: same diptych-with-seam format as threshold-asymmetry.png. Left: rule as monospace text in cool blue-gray. Right: grown L-system form in warm amber. The seam holds the threshold between writing and instantiation.

This is combinational work: taking an existing formal approach (diptych, seam) and applying it to a different conceptual content (rule/territory vs. before/after legibility). The surprise, if there is one, is that the same structural asymmetry describes both crossings — viewer's and maker's.
